29 /// Instances of the Type class are immutable and uniqued. They wrap a pointer
30 /// to the storage object owned by MLIRContext. Therefore, instances of Type
31 /// are passed around by value.
32 ///
33 /// Some types are "primitives" meaning they do not have any parameters, for
34 /// example the Index type. Parametric types have additional information that
35 /// differentiates the types of the same kind between them, for example the
36 /// Integer type has bitwidth, making i8 and i16 belong to the same kind by be
37 /// different instances of the IntegerType.
38 ///
39 /// Types are constructed and uniqued via the 'detail::TypeUniquer' class.
40 ///
41 /// Derived type classes are expected to implement several required
42 /// implementation hooks:
43 /// * Required:
44 /// - static bool kindof(unsigned kind);
45 /// * Returns if the provided type kind corresponds to an instance of the
46 /// current type. Used for isa/dyn_cast casting functionality.
47 ///
48 /// * Optional:
49 /// - static LogicalResult verifyConstructionInvariants(
50 /// Optional<Location> loc,
51 /// MLIRContext *context,
52 /// Args... args)
53 /// * This method is invoked when calling the 'TypeBase::get/getChecked'
54 /// methods to ensure that the arguments passed in are valid to construct
55 /// a type instance with.
56 /// * This method is expected to return failure if a type cannot be
57 /// constructed with 'args', success otherwise.
58 /// * 'args' must correspond with the arguments passed into the
59 /// 'TypeBase::get' call after the type kind.
60 ///
61 ///
62 /// Type storage objects inherit from TypeStorage and contain the following:
63 /// - The type kind (for LLVM-style RTTI).
64 /// - The dialect that defined the type.
65 /// - Any parameters of the type.
66 /// For non-parametric types, a convenience DefaultTypeStorage is provided.
67 /// Parametric storage types must derive TypeStorage and respect the following:
68 /// - Define a type alias, KeyTy, to a type that uniquely identifies the
69 /// instance of the type within its kind.
70 /// * The key type must be constructible from the values passed into the
71 /// detail::TypeUniquer::get call after the type kind.
72 /// * If the KeyTy does not have an llvm::DenseMapInfo specialization, the
73 /// storage class must define a hashing method:
74 /// 'static unsigned hashKey(const KeyTy &)'
75 ///
76 /// - Provide a method, 'bool operator==(const KeyTy &) const', to
77 /// compare the storage instance against an instance of the key type.
78 ///
79 /// - Provide a construction method:
80 /// 'DerivedStorage *construct(TypeStorageAllocator &, const KeyTy &key)'
81 /// that builds a unique instance of the derived storage. The arguments to
82 /// this function are an allocator to store any uniqued data within the
83 /// context and the key type for this storage.
84 class Type {
85 public:
86  /// Integer identifier for all the concrete type kinds.
87  /// Note: This is not an enum class as each dialect will likely define a
88  /// separate enumeration for the specific types that they define. Not being an
89  /// enum class also simplifies the handling of type kinds by not requiring
90  /// casts for each use.
91  enum Kind {
92  // Builtin types.
93  Function,
94  Opaque,
95  LAST_BUILTIN_TYPE = Opaque,
96 
97  // Reserve type kinds for dialect specific type system extensions.
98 #define DEFINE_SYM_KIND_RANGE(Dialect) \
99  FIRST_##Dialect##_TYPE, LAST_##Dialect##_TYPE = FIRST_##Dialect##_TYPE + 0xff,
100 #include "DialectSymbolRegistry.def"
101  };

119  bool operator!=(Type other) const { return !(*this == other); }

255  assert(impl && "isa<> used on a null type.");
256  return U::classof(*this);

259  return isa<U>() ? U(impl) : U(nullptr);
